# Standard Operating Procedure (SOP) - RFC 2119

When discussing SOP (Standard Operating Procedure), the following RFC 2119 key words define requirement levels:

## Requirement Levels

1. **MUST** / **REQUIRED** / **SHALL**
   - Absolute requirement of the specification
   - Non-negotiable, must be implemented

2. **MUST NOT** / **SHALL NOT**
   - Absolute prohibition of the specification
   - Never allowed under any circumstances

3. **SHOULD** / **RECOMMENDED**
   - Valid reasons may exist to ignore in particular circumstances
   - Full implications must be understood and carefully weighed before choosing a different course

4. **SHOULD NOT** / **NOT RECOMMENDED**
   - Valid reasons may exist when the behavior is acceptable or useful
   - Full implications should be understood and case carefully weighed before implementing

5. **MAY** / **OPTIONAL**
   - Truly optional item
   - Implementations with or without the option MUST interoperate
   - One vendor may include while another may omit

## Usage Guidelines

- Use these imperatives with care and sparingly
- MUST only be used where required for interoperation or to limit harmful behavior
- Must not be used to impose a particular method where not required for interoperability
- Consider security implications when not following MUST/SHOULD requirements
